# Contributing to VOIS_AICTE_Oct2025_Mohit-Bansal

We welcome contributions to our project! Please follow the guidelines below to ensure a smooth collaboration:

## How to Contribute
1. **Fork the repository**: You need your own copy of the repository to work on. Click the ‘Fork’ button on the top right.
2. **Clone your fork**: Use the command `git clone <your-fork-url>` to clone your forked repository to your local machine.
3. **Create a feature branch**: Before making any changes, create a new branch for your feature or bugfix. You can use the command `git checkout -b feature/your-feature-name`.
4. **Make your changes**: Implement your feature or bug fix. Make sure to add tests as appropriate.
5. **Commit your changes**: Use descriptive commit messages for your changes. For example: `git commit -m "Add new feature for ..."`
6. **Push to your fork**: Push your changes back to your forked repository using `git push origin feature/your-feature-name`.
7. **Create a pull request**: Go to the original repository and click on ‘New Pull Request’. Select your branch and submit the pull request.

## Code Style
- Follow the coding style and conventions used in the project.
- Make sure to run the tests and ensure your changes do not break any existing functionality.

## Issues
If you encounter a bug or want to propose a feature, please create an issue in the repository before implementing your changes.
